# Product Docs

This directory separates three things:

1. **What VS Code already is** — captured in [VS.md](./VS.md).
2. **What Typio wants to become** — captured in [vision.md](./vision.md).
3. **How we stage the transition** — to be captured in future stage and implementation docs.

## Current Docs

- [VS.md](./VS.md) — decomposes VS Code as an inherited product: technical systems, user experience, UI surfaces, workflows, risks, and opportunities.
- [vision.md](./vision.md) — Typio's product vision and redesign direction.
- [pi-agent-integration.md](./pi-agent-integration.md) — end-to-end plan for adding Pi as a first-class Agent Host provider.

## Rule of Thumb

If a note says **what VS Code already does**, put it in `VS.md`.

If a note says **what Typio should become**, put it in `vision.md`.

If a note says **how we will implement or sequence work**, put it in a planning/stages document such as `pi-agent-integration.md`.
